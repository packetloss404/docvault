Features

Document management,
taken to its logical conclusion.

21 domain-bounded apps. 14 processing plugins. Every feature built from real-world requirements across Lodestone, Mayan EDMS, and Paperless-ngx — then taken further.

01

14-Stage Processing Pipeline

Plugin architecture — drop a document in, get structured data out
Automation

Every document flows through an ordered chain of plugins: preflight validation, barcode detection, workflow triggers, pre-consume hooks, content parsing, language detection, date extraction, persistent storage, ML classification, zone OCR, AI analysis, NER, thumbnail generation, and post-consume hooks. Each plugin is independently toggleable.

02

OCR & Text Extraction

OCRmyPDF + Tesseract with six format-specific parsers

Multi-language OCR with configurable DPI (300 default), deskew, rotation, and three cleaning modes. PDF/A archive output. Dedicated parsers for PDF, images, Office documents, email (.eml/.msg), plain text, and ZIP archives. Three OCR modes: skip existing text, redo all, or force.

03

Workflow State Machine

Conditional transitions, escalation timers, AST-safe expression evaluator
Enterprise

Full state-machine engine with states, transitions, custom form fields, and 7 built-in action backends (set properties, add tags, move to cabinet, set metadata, launch sub-workflows, send email, fire webhooks). Conditions use a safe AST-based evaluator that whitelists only safe Python node types. Escalation timers auto-advance stalled documents.

ML Classification

Four MLP classifiers with TF-IDF and hash-based retraining
AI / ML

scikit-learn MLP neural networks auto-suggest tags, correspondents, document types, and storage paths. TF-IDF feature extraction with smart corpus hashing detects when retraining is needed. Confidence thresholds prevent low-quality suggestions. Fully automatic — learns from your manual classifications.

OpenAI + Ollama — ask your documents anything
AI / ML

LLM-powered document summarization, title suggestion, entity extraction, and conversational Q&A. FAISS vector store with sentence embeddings enables Retrieval-Augmented Generation across your entire corpus. Supports OpenAI cloud and Ollama local models — your data stays private if you want it to.

Full-Text & Semantic Search

Elasticsearch + FAISS vector search with facets and curations

Elasticsearch handles keyword search with faceted filtering, result highlighting, and permission-aware access control. FAISS vector search finds semantically similar documents using sentence embeddings. Reciprocal Rank Fusion merges both result sets. Built-in synonym management, search curations, click analytics, and 29 saved-view filter rule types.

Draw bounding boxes, extract structured fields from forms
Enterprise

Each zone extracts a specific field (invoice number, vendor name, total, date) using targeted OCR with configurable Tesseract PSM modes. Auto-matches templates to incoming documents with confidence scoring. Human review queue for low-confidence extractions.

13

Content-Addressable Storage

SHA-256 deduplication — identical files stored exactly once
Storage

Every file stored by its SHA-256 hash in a sharded path structure (ab/cd/full_hash). Duplicate uploads automatically reference the existing blob with incremented reference counting. Deletion only removes the physical file when the last reference is gone. Built-in integrity verification re-hashes stored files to detect corruption.

Security

Enterprise Security.
Zero Compromise.

OWASP Top 10 compliant, SOC 2 aligned, and GDPR-ready. From encryption at rest to legal hold, DocVault takes security as seriously as you do.

🔒

AES-256 Encryption at Rest

Every file encrypted with AES-256-CBC using PBKDF2 key derivation with 100,000 iterations. Random IV per file, chunked processing for memory efficiency.

🔐

SSO & Multi-Factor Auth

OpenID Connect, LDAP/Active Directory, OAuth2 providers (Google, Microsoft, GitHub). TOTP/HOTP two-factor authentication with pyotp.

📋

Comprehensive Audit Trail

Every model change logged with actor, target, and timestamp. The @audit_event decorator ensures nothing slips through. CSV/JSON export for compliance.

🛡

Role-Based + Object ACLs

Granular permission model: role-based access control, object-level ACLs via django-guardian, ownership semantics, and cabinet permission inheritance.

🌐

IP Access Control

Configurable IP whitelists and blacklists, rate limiting per user type (20/min anon, 120/min authenticated), CORS controls, and Content Security Policy headers.

📜

Document Signing (GPG)

GPG/PGP digital signatures for document integrity verification. Check-in/check-out locking prevents concurrent editing conflicts. WORM storage mode available.
