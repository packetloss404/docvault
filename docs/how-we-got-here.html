How DocVault got here

Lineage: comparative research anchored on Paperless-ngx, Mayan EDMS, and Lodestone, plus a 45+ system market scan for tier‑1 gaps beyond those three. Today’s repo: the root README.md is the live inventory—GPLv3 stack (Django 5.2, DRF 3.16, Django Channels, Celery 5.6 + Beat, Angular 21, Redis 7, optional Elasticsearch 8 + FAISS, S3/MinIO), OpenAPI at /api/docs/, Helm in deploy/, and a contributors welcome path via CONTRIBUTING.md + CODE_OF_CONDUCT.md.

About this page

One self-contained HTML file (no external scripts): a short visual story of where ideas came from and how the codebase is organized today. Numbers and stack details should track the root README.md over time.

// Timeline

Program narrative

Early milestones are research and spec anchors; the tail reflects what this repository documents today, not git archaeology.

Research phase Deep dives completed — Paperless-ngx (Angular + Django + ML barcodes), Mayan (ACLs, workflows, 100+ apps), Lodestone (non-destructive, MinIO, microservices).
Comparative lock Feature matrix + “adopt” lists — Explicit YES/NO/Our column; 33 adoption bullets grouped by source product.
Extended market 45+ system scan — E-sign flows, zone OCR, contributor portals, NER facets, legal hold patterns, etc. (Tier 1 features).
Vision → product draft Single-system mission — Combine Mayan workflows/security, Paperless ML/UX, Lodestone non-destructive + object storage posture; principles include modular monolith and AI-first (see historical product-spec where retained).
Build phase (this repo) Modular monolith + SPA — ~21 Django domain apps under docvault/ (AI, annotations, collaboration, documents, entities, esignatures, legal hold, ML, notifications, organization, physical records, portal, processing, relationships, search, security, sources, storage, workflows, zone OCR, …) plus Angular src-ui. Ongoing work is tracked in dev/work/CHECKLIST.md; see CONTRIBUTING.md to pick up a task, and dev/AGENTS.md for a maintainer-style map of the tree.
Delivery surface today Shipped themes (README) — Zone OCR, contributor portals + document requests, e-signatures (public + verified external signers), legal hold, CAS/dedup, physical records, relationship graph, hybrid + semantic search with analytics, WebSocket notifications, RBAC admin, and more—wired through DRF and OpenAPI docs at /api/docs/.
Community Contributors welcome — CONTRIBUTING.md, CODE_OF_CONDUCT.md, issue templates under .github/, private disclosure via SECURITY.md.

What each lane contributed

Research intent mapped to README-listed capabilities in this codebase.

Paperless-ngx

  • Angular SPA + Django + DRF as the default OSS DMS UX spine (now Angular 21 + Bootstrap 5.3 in src-ui).
  • ML classification, barcodes/ASN, hybrid + semantic search (FAISS), saved views, global command palette, LLM document chat/summarize/entities, Redis-backed ML caches.

Mayan EDMS

  • Enterprise workflows (state machines, triggers, escalations), object ACLs via django-guardian, cabinets, retention, encryption-at-rest option, LDAP/OIDC/social auth + 2FA, quotas, audit logging, GPG signing.

Lodestone

  • Non-destructive originals + configurable paths; S3/MinIO object storage; background processing realized with Celery workers/Beat instead of microservice sprawl.

45+ market scan

  • Tier‑1 deltas reflected in apps such as esignatures/, zone_ocr/, portal/, legal_hold/, entities/, plus search analytics/synonyms/curations—beyond what the three anchors alone provided.
// Architecture

Research → repo

Why the codebase is shaped as a modular monolith with a separate SPA.

  • Rejected as default: Lodestone-style microservices sprawl for core team velocity.
  • Rejected as default: Mayan-scale 100+ app explosion for a greenfield codebase.
  • Chosen: Modular monolith (clear Django app boundaries) + Angular SPA + Celery workers + optional Elasticsearch + FAISS + Redis + Channels—documented in README.md and embodied under docvault/ + src-ui/.
  • Ship & observe: Production Docker + Helm under deploy/; health/readiness/metrics endpoints per README API table.

Historical research — Comparative notes, per-product deep dives, and extended market tiers may appear under dev/archive/original-spec/research/ in copies of this repository that ship that bundle.

What to trust for “today” — Root README.md (features, stack, tests, layout), plus CONTRIBUTING.md for how to contribute.
