Timeline (program narrative)

Early milestones are research and spec anchors; the tail reflects what this repository documents today (see README.md), not git archaeology.

Research phase Deep dives completed — Paperless-ngx (Angular + Django + ML barcodes), Mayan (ACLs, workflows, 100+ apps), Lodestone (non-destructive, MinIO, microservices).
Comparative lock Feature matrix + “adopt” lists — Explicit YES/NO/Our column; 33 adoption bullets grouped by source product.

Vision → product draft Single-system mission — Combine Mayan workflows/security, Paperless ML/UX, Lodestone non-destructive + object storage posture; principles include modular monolith and AI-first (see historical product-spec where retained).
Build phase (this repo) Modular monolith + SPA — ~21 Django domain apps under docvault/ (AI, annotations, collaboration, documents, entities, esignatures, legal hold, ML, notifications, organization, physical records, portal, processing, relationships, search, security, sources, storage, workflows, zone OCR, …) plus Angular src-ui; agent-oriented execution in dev/work/CHECKLIST.md per dev/AGENTS.md.

What each lane contributed (summary)

Below ties research intent to README‑listed capabilities in this codebase.

Paperless-ngx

  • Angular SPA + Django + DRF as the default OSS DMS UX spine (now Angular 21 + Bootstrap 5.3 in src-ui).
  • ML classification, barcodes/ASN, hybrid + semantic search (FAISS), saved views, global command palette, LLM document chat/summarize/entities, Redis-backed ML caches.

Mayan EDMS

  • Enterprise workflows (state machines, triggers, escalations), object ACLs via django-guardian, cabinets, retention, encryption-at-rest option, LDAP/OIDC/social auth + 2FA, quotas, audit logging, GPG signing.

Lodestone

  • Non-destructive originals + configurable paths; S3/MinIO object storage; background processing mental model realized with Celery workers/Beat instead of microservice sprawl.

45+ market scan

  • Tier‑1 deltas now reflected in apps such as esignatures/, zone_ocr/, portal/, legal_hold/, entities/, plus search analytics/synonyms/curations—beyond what the three anchors alone provided.

Architecture decision (research → repo)

  • Rejected as default: Lodestone-style microservices sprawl for core team velocity.
  • Rejected as default: Mayan-scale 100+ app explosion for a greenfield codebase.
  • Chosen: Modular monolith (clear Django app boundaries) + Angular SPA + Celery workers + optional Elasticsearch + FAISS + Redis + Channels—documented as the operational stack in README.md and embodied under docvault/ + src-ui/.
  • Ship & observe: Production Docker + Helm chart under deploy/; health/readiness/metrics endpoints per README API table.
